Developer Calculation Logic & Architecture Notes

This page documents every metric shown in the dashboard, how it should be calculated, data sources, and implementation notes. All calculations assume data is pulled from GHL webhooks + AI conversation logs stored in your database.

1. SMS Delivery & Response Metrics

1.1 SMS Delivery Rate

Delivery Rate = (Messages Successfully Delivered / Messages Sent) x 100

Source: Carrier delivery receipts via GHL webhook. Track status: "delivered" vs status: "failed" / "undelivered". Group by subaccount_id and timeframe. Update every 15 minutes.

1.2 SMS Response Rate

Response Rate = (Unique Leads Who Replied / Messages Delivered) x 100

Only count first reply per lead per campaign window. Exclude STOP/OPTOUT from response count (those go to Opt-Out Rate).

1.3 Bookings Per 100 Leads Engaged

Bookings / 100 = (Appointments Booked / Leads Who Received at least 1 Message) x 100

"Engaged" = lead received at least 1 outbound message in the timeframe window. Appointment booked = GHL calendar event created with status confirmed.

2. Opt-Out, DNC & Compliance

2.1 Opt-Out Rate

Opt-Out Rate = (Opt-Out Responses / Messages Delivered) x 100

Opt-out keywords: STOP, UNSUBSCRIBE, CANCEL, END, QUIT (case-insensitive).

NOTE: A response that is ONLY an opt-out keyword should NOT count toward the positive Response Rate. Separate field: opt_out_as_only_response tracked per lead.

2.2 DNC Rate

DNC Rate = (Leads Flagged DNC / Total Leads Messaged) x 100

DNC flags come from: (a) explicit STOP reply, (b) manual DNC flag in GHL contact, (c) federal/state DNC list match.

3. AI Accuracy Scoring

3.1 Composite Score Formula

AI Accuracy % = (QC Manual Score x 0.70) + (AI Self-Score x 0.30)

QC Manual Score: Each conversation reviewed in QC Portal is marked Approved (100pts), Modified (60pts), or Failed (0pts). Average across all reviewed conversations in the rolling 7-day window.

AI Self-Score: After each conversation closes, run a secondary Claude analysis prompt. Average those scores across all conversations in the window.

3.2 Good / OK / Wrong Breakdown

Good: AI Self-Score >= 80 AND not failed in QC
OK: AI Self-Score 50-79 OR modified in QC
Wrong: AI Self-Score < 50 OR failed in QC
IMPORTANT: Manual QC overrides AI self-score. If a message is manually failed in QC, it is classified as "Wrong" regardless of AI self-score.

4. Time Saved Calculations

4.1 SMS Responded To

Time Saved = (Total Outbound SMS Sent + Total Inbound Replies Handled by AI) x 100 seconds / 3600

Rationale: 100 seconds = estimated time for a human agent to read, compose, personalize, and send one SMS reply.

4.2 Bulk Campaigns Scheduled & Sent

Time Saved = Number of Campaigns Scheduled x 8 minutes per campaign / 60

4.3 Phone Calls

Call Time Saved = Calls Routed by AI (not escalated) x 2 minutes / 60

4.4 Admin (Per Contact)

Admin Saved = Total Contacts Engaged x 15 seconds / 3600

5. Cost Metrics

5.1 AI Token Cost

Token Cost = (Input Tokens x Input Rate) + (Output Tokens x Output Rate)

Pull token usage from Anthropic API response headers. Store per conversation_id. Aggregate by subaccount + timeframe.

5.2 SMS Carrier Cost

Carrier Cost = Segments Sent x $0.0075 per segment

5.3 Cost Per Message (Blended)

Cost Per Message = (Total AI Cost + Total Carrier Cost) / Total Messages Sent

5.4 Avg Segments Per Message

Avg Segments = Sum of All Segment Counts / Total Messages Sent
Alert threshold: if avg segments > 1.6, flag for review.

6. Pipeline Stage Logic

Pipeline stages are defined in GHL as custom pipeline stages. Each lead has a pipeline_stage_id. Refresh every 30 minutes.

Pipeline Stages (in order): 1. Raw Lead / Uncontacted 2. AI Outreach Initiated 3. First Response Received 4. Engaged -- Qualifying 5. Needs Identified 6. Quote / Presentation Sent 7. Appointment Scheduled 8. Appointment Confirmed 9. Appointment Completed 10. Application Started 11. Application Submitted 12. Pending Underwriting 13. Policy Issued 14. Cross-Sell Pipeline 15. Long-Term Nurture 16. Aged -- Re-Engage Queue 17. Disqualified 18. DNC / STOP Received 19. Lost -- No Response

7. 8-Week Rolling Average

Rolling Avg (Week N) = (Qualified Leads in Week N / Total Leads Engaged in Week N) x 100

Run this calculation independently for fresh and aged leads. Recalculate each Monday at midnight.

8. Wordtrack / Message Performance

Each message sent must carry a message_id and message_version in metadata.

  • Response Rate: replies to that specific message / times that message was sent
  • Opt-Out Rate: opt-out replies to that message / sends
  • Booking Rate: conversations where that message was last sent before a booking occurred / sends
  • Transfer Rate: same but for live transfer outcome

9. Response Outcome Categories

  • Booked: conversation ended with GHL appointment created
  • Live Transfer: conversation ended with transfer flag set
  • Opted Out: STOP/UNSUBSCRIBE received
  • No Action (Fluff): responded but no qualifying progress
  • Pending: conversation is open, awaiting reply
  • No Response: no reply within 72-hour window

10. Sandbox Mode Architecture

Manual mode: human types replies, system sends to production Claude endpoint. All sandbox conversations are tagged environment: "sandbox" and do NOT affect production metrics.

11. Random Conversation Pull (QC)

Pull Random: SELECT conversation_id FROM conversations WHERE status = 'closed' AND qc_reviewed = false ORDER BY RANDOM() LIMIT 1

Pull from last 14 days. Weight toward conversations with AI confidence score < 70.
